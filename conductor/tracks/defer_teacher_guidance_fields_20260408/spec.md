# Specification: Document Deferral of Teacher Guidance Fields

## Overview
The curriculum currently has 5 teacher guidance fields (preparation, facilitationGuidance, checksForUnderstanding, watchFors, nextSteps) defined in the DailyLesson type, but none are populated across the 80 daily lessons. This track documents the decision to defer populating these fields and formally scopes the work for a future track.

## Functional Requirements
- Update current_directive.md to record the deferral decision
- Add a tech-debt.md entry for the unpopulated teacher guidance fields
- Create a future work scope document outlining what's needed to populate the fields
- Update tracks.md to mark this track as active

## Non-Functional Requirements
- The documentation must be clear and actionable for a future implementation
- The deferral decision must be aligned with the current project priorities

## Acceptance Criteria
- [ ] current_directive.md updated to reflect the deferral decision
- [ ] tech-debt.md includes a new entry for the unpopulated teacher guidance fields
- [ ] A future work scope document is created in the track directory
- [ ] tracks.md is updated with the new track

## Out of Scope
- Populating any teacher guidance fields across the 80 daily lessons
