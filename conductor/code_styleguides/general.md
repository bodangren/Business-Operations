# General Code Style Principles

## Readability
- Code should be easy to read and understand by humans
- Avoid overly clever or obscure constructs
- Use meaningful variable and function names

## Consistency
- Follow existing patterns in the codebase
- Maintain consistent formatting, naming, and structure
- Use component patterns from AGENTS.md

## Simplicity
- Prefer simple solutions over complex ones
- Break down complex problems into smaller, manageable parts

## Maintainability
- Write code that is easy to modify and extend
- Minimize dependencies and coupling

## Documentation
- Document *why* something is done, not just *what*
- Keep documentation up-to-date with code changes
- Document new components in MCP knowledge base per AGENTS.md

## Project-Specific Conventions
- Default exports for interactive learning components
- Named exports for UI primitives (PhaseHeader, PhaseFooter, Badge, Card)
- TypeScript interfaces from `src/types/unit.ts`
- Tailwind CSS v4 classes
