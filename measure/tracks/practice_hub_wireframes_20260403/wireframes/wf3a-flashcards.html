Annotation — WF3a Key Decisions: Card shows bilingual term on front (English large, Chinese below). Back shows bilingual definition. "Review Again" cards get re-queued to the end of the session. Phase 3 — Actions, Data & Behavior: Actions: • Tap card or Space → flip card (CSS transform toggle) • "Review Again" or ← key → mark term as needs-review, re-queue to end, increment attempt count • "Got It!" or → key → mark term as mastered for this session, advance counter, update mastery score • Session complete → show score card with: correct count, total, accuracy %, time spent • "Try Matching" → /student/practice-hub/matching?unit={id} • "Take Practice Test" → existing practice test route • "View in Glossary" → /student/glossary#{termId} • "Back to Unit" → /student/unit{unitId} Required Data: • termList[] — from glossary-data.ts filtered by unitId, shuffled at session start • Each term: { termId, termEn, termZh, definitionEn, definitionZh, topic } • sessionState — in-memory: { currentIndex, reviewQueue[], completed[], startTime } • mastery update — localStorage key: practice_terms_{termId} = { masteryScore, lastReviewed, attempts, bestStreak } Local-Only: • Session state is ephemeral (in-memory only, lost on page refresh) • Mastery scores are persisted to localStorage on each "Got It" / "Review Again" click • No server involvement — entire session is client-side Static Constraint: Cards render from a static term list. The flip animation is pure CSS. No dynamic data fetching during session. Export Payload Relevance: Session results contribute to export data: { mode: "flashcards", unitId, termCount, score, date, duration } Stored in practice_sessions[] localStorage for export aggregation.
